## 變更背景

`src/app/[locale]/` 是整個項目中**規模最大、結構最複雜的目錄**，包含 76 個 page.tsx、20 個 admin 模組、3 層布局層級，卻是唯一沒有 CLAUDE.md 導航文檔的主要目錄。

### 問題量化

| 指標 | 數量 |
|------|------|
| 頁面文件 (page.tsx) | 76 個 |
| Admin 模組子目錄 | 20 個 |
| Layout 層級 | 3 層（Root → Auth/Dashboard） |
| 路由組 | 2 個（`(auth)` + `(dashboard)`） |
| Client 組件 | 2 個（audit/query、companies/review） |
| 測試工具頁面 | 3 頁 + 13 個測試組件 |

### 痛點

1. **導航困難** — AI 助手開發新頁面時需逐層探索目錄找到正確位置
2. **慣例不明確** — 20 個 admin 模組的 CRUD 路由模式未統一文檔化
3. **頁面 ↔ API ↔ 組件 ↔ i18n 對應關係分散** — 沒有集中索引
4. **路由組和布局繼承邏輯隱含** — `(auth)` vs `(dashboard)` 的區別需要查看 layout.tsx 才知道

---

## 變更目標

- **建立路由結構導航文檔**：讓 AI 助手和開發者能在 2 分鐘內找到任何頁面的位置
- **文檔化 Admin 模組索引**：20 個模組的完整清單、路由模式、對應 API 和 i18n 命名空間
- **說明路由組和布局層級**：清楚解釋 `(auth)` / `(dashboard)` 分組規則
- **提供新增頁面的標準流程**：確保新頁面遵循既有慣例
