  重點：一個登入帳號被建立，不代表自動就有一個資料庫、也不代表有建表權限。這三者要分開準備。容器啟動時要建 122
  張表，所以必須先有那個資料庫 + 帳號對它有權限。

  最乾淨的解法（就是你之前選的「DBA 建 DB + 給 role 完整權限」）

  請 DBA（Andy.kao，用 Entra admin 身分連線）執行一行 SQL，一次解決「建 DB + 給權限」：

  CREATE DATABASE ai_document_extraction OWNER raposcmaidocprocessingdev;

  把 raposcmaidocprocessingdev 設為這個 DB 的 owner，它就自動擁有完整權限（含在 public schema 建表，PG 18 適用）。容器啟動 bootstrap
  就能成功建表。

  ▎ 若 DBA 想用 Portal/az 先建空 DB，那改下這兩行補權限即可：
  ▎ ALTER DATABASE ai_document_extraction OWNER TO raposcmaidocprocessingdev;

  為什麼不我自己建？

  我用 Contributor 可以從控制面建空 DB（az postgres flexible-server db create），但建出來的 DB owner
  會是伺服器管理員，raposcmaidocprocessingdev 仍沒有建表權限——那個 GRANT/OWNER 變更需要以 DB 管理員身分連進去做，而我連不到私有
  DB。所以由 DBA 跑上面那行最省事。
