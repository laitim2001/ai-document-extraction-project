  4. AI Session Start Protocol（EKP §10.3）

  EKP 強制每個 session 開始時 AI 必須順序執行 6 步，再 reply 用戶第一句：
  1. 讀 SITUATION 啟動文件
  2. 讀 active phase plan.md
  3. 讀 active phase checklist.md
  4. 讀 active phase progress.md 最近 3 個 Day-N entries
  5. Run git status --short + git log --oneline -5
  6. 不清楚就問

  特別有價值：「Compact 後特殊處理」 — /compact 觸發後 context 重組，AI 必須重讀步驟 1-4，因為 compact 對 standing instructions
  retention 只有 ~60%。

  本項目現狀：有 SITUATION-1 但沒有強制執行協議；沒有 compact 後 recovery 規則（這是踩過坑才會知道的痛點）。

  可參考：把 SITUATION-1 升級為「強制啟動協議」+ 加「compact 後 re-read 規則」。

  ├────────────┼────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ 與檔案無關 │ 認證、權限、檔案類型、大小全部通過，純粹卡在寫入 blob 那一步，任何檔案都會失敗                                     │
  ├────────────┼────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ 修復       │ docker-compose.yml 的 azurite 啟動參數加 --skipApiVersionCheck（Azurite 官方為此情況設計的 flag），並重建容器      │
