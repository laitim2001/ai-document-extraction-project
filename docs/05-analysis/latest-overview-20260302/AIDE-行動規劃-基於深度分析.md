## 十、決策要點

在開始執行前，建議確認以下決策：

1. **P0-2 策略選擇**: 在 middleware 全局攔截 vs 逐路由添加認證？
   - 全局攔截更安全但需維護白名單
   - 逐路由更精細但工作量大

2. **P1-3 日誌框架選擇**: pino vs winston vs 自建？
   - pino: 性能最佳，Next.js 生態兼容
   - winston: 功能最全，社區最大

3. **P2-4 消息隊列選擇**: Azure Service Bus vs BullMQ (Redis)?
   - Azure Service Bus: 與現有 Azure 生態統一
   - BullMQ: 利用現有 @upstash/redis，輕量

4. **執行模式**: 串行 vs 並行 Agent?
   - P0 項目之間有依賴，建議串行
   - P1 項目部分可並行（如 P1-1 測試 與 P1-5 CI/CD）
